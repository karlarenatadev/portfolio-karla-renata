target O Problema de Negócio

No cenário comercial competitivo, as equipas de vendas perdem muito tempo e energia com negociações ("deals") que têm baixa probabilidade de fechar. O desafio deste projeto foi criar um modelo preditivo capaz de analisar o comportamento histórico e responder a uma pergunta de milhões: Qual é a probabilidade real deste negócio ser ganho ou perdido?

psychology A Abordagem Técnica (Machine Learning)

A inovação do projeto reside em converter dados descritivos de um CRM em estatística acionável. Para lidar com o desafio clássico do desbalanceamento de dados em vendas (onde a maioria das vendas falha e poucas têm sucesso), implementei:

  • Feature Engineering: Utilização de Target Encoding para capturar a performance histórica de variáveis categóricas.
  • Balanceamento (SMOTE): Aplicação do método Synthetic Minority Over-sampling Technique para equilibrar as classes minoritárias, evitando que o modelo ficasse enviesado.
  • Modelagem: Treino e otimização de um algoritmo Gradient Boosting, que aumentou o ROC-AUC em quase 20% em relação ao modelo base de regressão logística.

Snippet do Pipeline de Treinamento:

# Aplicando SMOTE para balanceamento e treinando o Gradient Boosting
from imblearn.over_sampling import SMOTE
from sklearn.ensemble import GradientBoostingClassifier

smote = SMOTE(random_state=42)
X_train_resampled, y_train_resampled = smote.fit_resample(X_train, y_train)

gb_model = GradientBoostingClassifier(n_estimators=100, learning_rate=0.1, max_depth=5)
gb_model.fit(X_train_resampled, y_train_resampled)

# Gerando o Score de Risco (Probabilidade em %)
y_proba = gb_model.predict_proba(X_test)[:, 0] * 100

dashboard A Entrega de Valor (Power BI)

Um modelo de Machine Learning a correr num terminal escuro não gera impacto comercial. A verdadeira mágica acontece na entrega. Os resultados gerados (Scores de Risco) foram injetados num Dashboard Interativo no Power BI.

Com esta solução, o gestor de vendas pode agora aplicar filtros dinâmicos, visualizar um gráfico de dispersão com os riscos inerentes de cada negociação e focar os esforços imediatos da equipa exatamente nos negócios sinalizados como "Risco Crítico" (utilizando DAX e formatação condicional avançada).
