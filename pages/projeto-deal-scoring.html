🎯 O Problema de Negócio

No cenário comercial, equipas de vendas perdem muito tempo e energia com negociações ("deals") que têm baixa probabilidade de fechar. O desafio deste projeto foi criar um modelo preditivo capaz de analisar o comportamento histórico e responder: Qual é a probabilidade real deste negócio ser ganho?

🧠 A Abordagem Técnica

A inovação do projeto reside em converter dados descritivos de um CRM em estatística acionável. Para lidar com o desafio clássico do desbalanceamento de dados em vendas, implementei:

  • Feature Engineering: Utilização de Target Encoding para capturar a performance histórica de variáveis categóricas.
  • Balanceamento: Aplicação do método SMOTE (Synthetic Minority Over-sampling Technique) para equilibrar as classes minoritárias.
  • Modelagem: Treino e otimização de um algoritmo Gradient Boosting, que aumentou o ROC-AUC em quase 20% em relação ao baseline.

Snippet do Pipeline de Machine Learning:

# Aplicando SMOTE para balanceamento e treinando o Gradient Boosting from imblearn.over_sampling import SMOTE from sklearn.ensemble import GradientBoostingClassifier smote = SMOTE(random_state=42) X_train_resampled, y_train_resampled = smote.fit_resample(X_train, y_train) gb_model = GradientBoostingClassifier(n_estimators=100, learning_rate=0.1, max_depth=5) gb_model.fit(X_train_resampled, y_train_resampled) # Gerando o Score de Risco (Probabilidade) y_proba = gb_model.predict_proba(X_test)[:, 0] * 100

📊 A Entrega de Valor (Power BI)

Um modelo de ML a correr num ecrã preto não gera impacto comercial. A verdadeira mágica acontece na entrega. Os resultados (Scores de Risco) foram consumidos por um Dashboard Interativo no Power BI.

Agora, o gestor de vendas pode aplicar filtros dinâmicos, visualizar um gráfico de dispersão com os riscos de cada negociação e focar os esforços da equipa exatamente nos negócios sinalizados como "Risco Crítico" (com formatação condicional) hoje.
